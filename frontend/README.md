# 📖 About Kite Console

Kite Console is a production-inspired full-stack stock trading platform that recreates the architecture and workflows of a modern online brokerage system.

Unlike UI-focused trading clones, this project emphasizes backend engineering, transactional consistency, secure authentication, asynchronous processing, and automated financial workflows. Every major module has been designed to simulate how brokerage platforms manage user funds, trading activity, settlements, and portfolio updates behind the scenes.

The application consists of three independent layers working together:

- A modern **Marketing Website** introducing the platform.
- A secure **Trading Dashboard** for authenticated users.
- A centralized **Express.js API** responsible for business logic, trading operations, authentication, and scheduled background services.

Built with React, Express.js, MongoDB Atlas, and Node.js, the platform demonstrates practical implementation of secure full-stack architecture while maintaining a clean and responsive user experience.

---

# 🎯 Why This Project?

Most stock trading clones focus primarily on replicating the appearance of platforms such as Zerodha Kite. Kite Console takes a different approach.

Instead of stopping at the user interface, this project recreates many of the backend workflows that power real brokerage systems, including:

- Automated order processing
- Capital locking during pending trades
- Portfolio reconciliation
- Scheduled end-of-day settlement
- Background order matching
- Transaction-safe database operations
- Secure cross-origin authentication
- Financial ledger management

The objective was not simply to build another trading dashboard, but to understand and implement the engineering concepts involved in developing scalable financial applications.

# 🏗️ Architecture Overview

Kite Console follows a modular **three-tier architecture** that separates the presentation layer, business logic, and data storage into independent components.

The platform consists of two React applications communicating with a centralized Express.js backend. The backend manages authentication, order execution, portfolio calculations, payment integration, scheduled background services, and database interactions, while MongoDB Atlas provides persistent storage for user accounts, portfolios, orders, and transactions.

This architecture promotes maintainability, scalability, and a clear separation of responsibilities, making the application easier to extend and maintain.

---

                                                     │
                                    Price Sync • Order Matching

## ✨ Architectural Highlights

- Independent Marketing Website and Trading Dashboard.
- Centralized Express.js REST API serving multiple client applications.
- Modular MVC architecture with dedicated controllers, services, middleware, and models.
- MongoDB Atlas used for persistent transactional storage.
- JWT authentication protecting all secured endpoints.
- Background schedulers managing price synchronization, order matching, and settlement.
- Razorpay Test Checkout integrated for wallet deposits.
- Stateless client-server communication enabling future scalability.

# 📈 Engineering Highlights

Beyond implementing a responsive trading interface, Kite Console focuses on recreating the backend workflows commonly found in modern brokerage systems. The following engineering decisions were made to improve consistency, reliability, and maintainability throughout the application.

---

## 💰 Preventing Double Spending

### Challenge

Users could unintentionally spend the same funds or shares by placing multiple pending orders simultaneously.

### Solution

When a limit order is submitted, the required cash or shares are immediately transferred into a blocked balance before the order enters the matching queue.

### Result

- Eliminates duplicate spending.
- Preserves portfolio consistency.
- Simulates real brokerage fund reservation.

---

## ⚡ Background Order Matching Engine

### Challenge

Limit orders should execute only after the market reaches the requested price.

### Solution

A scheduled background service periodically compares pending orders against market prices and automatically executes qualifying trades.

### Result

- Fully automated order execution.
- Clean separation between user requests and execution logic.
- Easily extendable for live market integrations.

---

## 📊 End-of-Day Settlement Engine

### Challenge

Intraday positions must be converted into long-term holdings after market close.

### Solution

A scheduled settlement process aggregates active positions, calculates weighted average purchase prices, and migrates them into permanent holdings.

### Result

- Accurate portfolio tracking.
- Simplified long-term investment management.
- Consistent asset lifecycle.

---

## 🔄 Active-on-Wake Recovery

### Challenge

Free hosting providers suspend inactive backend services, causing scheduled jobs to be missed.

### Solution

Whenever the application becomes active again, the backend validates pending settlement windows and executes any missed operations before serving portfolio data.

### Result

- Prevents inconsistent portfolio states.
- Ensures eventual data consistency.
- Improves reliability on free-tier deployments.

---

## 🔐 Cross-Origin Session Synchronization

### Challenge

The Marketing Website and Trading Dashboard run on separate origins but should behave as a single authenticated platform.

### Solution

Session tokens are synchronized across both applications, ensuring that logging out from one immediately invalidates the other.

### Result

- Consistent authentication experience.
- Improved account security.
- Seamless Single Sign-Out workflow.

---

## 🗄 Transaction-Safe Database Operations

### Challenge

Financial operations often update multiple collections simultaneously. Partial updates could leave inconsistent data.

### Solution

Critical workflows execute within MongoDB transactions, ensuring every related database operation either completes successfully or rolls back entirely.

### Result

- Improved data integrity.
- Reliable financial operations.
- Consistent portfolio records.
